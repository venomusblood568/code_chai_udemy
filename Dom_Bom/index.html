Form Handling

How you will caught label cause both have same name.
answer => i think i can cause one is using id and one is using for
Why This Works:
id="feedbackInput" uniquely identifies the input field.
for="feedbackInput" links the label to the input using the input's id.
They are not conflicting because their roles are distinct: id targets an actual element in the DOM. for refers to the element with the matching id for accessibility. 

    Catching Each Element:
    
    To target the label:
    const label = document.querySelector('label[for="feedbackInput"]');
    console.log(label.textContent); // Outputs: "Your Feedback:"
    
    To target the input:
    const input = document.getElementById('feedbackInput');
    console.log(input.placeholder); // Outputs: "Enter your Feedback"
